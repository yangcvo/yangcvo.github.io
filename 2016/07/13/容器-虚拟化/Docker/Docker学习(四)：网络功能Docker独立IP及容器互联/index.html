Docker 独立IP及容器互联

内置bridge (nat)

  • 缺点：

需要配置服务注册/发现，否则宿主机端口分配困难，容易冲突。

由于每个容器暴露的端口都不一致，造成前端路由层nginx配置（proxy_pass）里面无法使用dns方式。

端口映射要在容器启动时指定好，后期无法变更。

测试发现nat不支持websocket.

自建桥接网络

  • 优点：

每个容器都有自己的独立IP，对外提供服务，如：nginx+php ，nginx-resin ，都可以使用默认的80端口。

由于容器保罗端口都可以使用80端口，因此前端路由层nginx配置（proxy_pass）里可以使用dns方式。

无需为后期端口映射添加而烦恼
