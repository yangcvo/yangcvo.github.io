Ansible + Jenkins + gitlab搞定自动发布

2016年9月12日

用Ansible + Jenkins搞定自动发布

Ansible是相对简单的批量管理工具，支持模板管理等高级功能。搞定了自动发布，开发的服务器需求已经明显下降，只要把代码提交到 Git主干，就会自动触发发布。

今天有空就简单记录下用Ansible + Jenkins搞定自动发布 。这篇是主要环境安装。

系统环境：centos7.1

第一步搭建ansibles

可以参考我之前写的ansibles学习篇。

第二步安装Jenkins环境

安装配置tomcat
安装配置Java
安装配置maven
安装配置Jenkins

这里我指定一个路径方便后期自己维护。

cd /srv/

去官网下载包资源：

apache-tomcat-8.0.28.tar.gz
jenkins.war
jdk1.8.0_66.tar.gz
maven.tar.gz

没有下载成功的我这里已下载好了。最新版本jenkins.war
tomcat.tar.gz
maven.tar.gz

tar -zxvf apache-tomcat-8.0.28.tar.gz
mv apache-tomcat-8.0.28 tomcat
useradd -s /sbin/nologin tomcat 
chown -R tomcat:tomcat tomcat

tar -zxvf jdk1.8.0_66.tar.gz

这里在配置环境变量。

vim /etc/profile.d/java.sh
export JAVA_HOME=/srv/jdk1.8.0_66
export CLASS_PATH="$JAVA_HOME/lib:$JAVA_HOME/jre/lib"
export PATH=$PATH:$JAVA_HOME/bin

然后解压maven

tar -zxvf maven.tar.gz

这里定义你有了maven仓库。这里需要自己编写。

在然后把Jenkins放进去重启自动解压。

mv jenkins.war /srv/tomcat/webapps/.

cd /tomcat
 ./bin/startup.sh

启动tomcat查看端口。

[root@docker1 bin]# netstat -ntulp

新版本会提示下载安装插件。下载以后会让你设置账号和密码还有邮箱。

如何构建这个过程；打通gitlab，拉取代码自动构建war包。 这个我已经在github已经写了。

后面就是结合ansible-Playbook基本语法 写控制发布服务器，然后在jenkins 上面写ansible 发布脚本&命令。

现在我们Git使用的是 GitLab，同时为了安全我们做了一层LDAP代理，效果相当于“将军令”，操作机、Git和Jenkins用 OpenLDAP 做统一认证，后续用到的Redmine、Grafana、Zabbix 等都接入了OpenLDAP认证，每个人都有个动态口令，每次验证都需要用到。
